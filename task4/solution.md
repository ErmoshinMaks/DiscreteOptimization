# Домашнее задание 4: Facility Location

## Используемая идея

Точное решение этой задачи NP-трудно, поэтому реализован быстрый эвристический решатель.

### 1) Конструктивный жадный старт

Клиенты сортируются по убыванию спроса.  
Для каждого клиента выбирается допустимый магазин с минимальной **инкрементальной** стоимостью:

- если магазин уже открыт: только расстояние;
- если магазин закрыт: расстояние + амортизированная часть стоимости открытия

s_f * d_c / cap_f, где  
  
`s_f` — полная стоимость открытия магазина `f`,

`d_c` — спрос клиента `c`,

`cap_f` — вместимость магазина `f`.

Это быстро строит допустимое начальное решение.

### 2) Локальное улучшение 1: перераспределение между открытыми магазинами

После старта выполняется проход по клиентам в случайном порядке:

- для каждого клиента ищется лучший открытый магазин среди списка ближайших (топ-K),
- перенос делается только если емкость позволяет и целевая функция уменьшается.

### 3) Локальное улучшение 2: открытие нового магазина

Периодически рассматриваются случайные закрытые магазины-кандидаты:

- для каждого оценивается выигрыш при переносе в него части клиентов,
- перенос выбирается жадно по максимальной экономии расстояния в пределах емкости,
- магазин открывается только если суммарная экономия превышает его стоимость открытия.

### 4) Локальное улучшение 3: закрытие невыгодного магазина

Пробуем закрыть дорогие открытые магазины:

- временно перераспределяем всех их клиентов в другие открытые магазины,
- если это возможно по емкостям и прирост расстояния меньше стоимости закрываемого магазина — закрытие принимается.

### 5) Multi-start

Алгоритм запускается несколько раз с разными seed и случайным порядком клиентов в жадном старте.  
Итогом выбирается лучшее найденное решение.

## Сложность

- Построение матрицы расстояний: O(MN).
- Жадный старт: O(MN).
- Локальные итерации: зависят от числа проходов, но каждый шаг ограничен списком ближайших магазинов.
