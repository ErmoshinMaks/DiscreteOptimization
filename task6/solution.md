# Домашнее задание 6: VRP

## Реализованный метод

### 1) Конструктивные старты (два типа)

В мультистарте чередуются несколько построений:

- **Clarke-Wright Savings**: начинаем с маршрута на каждого клиента и сливаем маршруты по убыванию savings  
`s(i,j) = dist(0,i) + dist(0,j) - dist(i,j)`  
при соблюдении ограничений на концы маршрутов и емкость.
- **Sweep**: сортируем клиентов по углу относительно депо (с случайным сдвигом и направлением), затем режем последовательность по емкости.
- **Sweep + DP split**: после угловой сортировки разбиение последовательности на маршруты делается динамикой (минимальная стоимость разбиения при ограничениях емкости и лимите машин).
- **Route-first / split-second**: строится общий тур по клиентам (nearest-neighbor + 2-opt), затем этот порядок также режется динамикой на емкостные маршруты.

### 2) Локальный поиск

На каждом старте выполняются улучшения:

- **2-opt внутри маршрута** (убирает пересечения и сокращает длину);
- **Relocate между маршрутами** (перенос одного клиента при соблюдении емкости и отрицательной дельте цели);
- **Swap между маршрутами** (обмен клиентов между двумя маршрутами, если цель уменьшается).

### 3) Мультистарт и ограничение времени

Запускается несколько независимых стартов с разными seed.  
Общий бюджет времени (`VRP_TIME`, по умолчанию `560` секунд) делится между оставшимися стартами, сохраняется лучшее допустимое решение.

## Что пробовалось и почему оставлен этот вариант

1. **Только Savings без альтернативных стартов**
  Работает стабильно, но на ряде инстансов проигрывает при неудачном порядке слияний.
2. **Только Sweep**
  Быстро и часто дает неплохую геометрию, но сильнее зависит от углового разреза и чаще застревает в худших локальных минимумах.
3. **Комбинация Savings + Sweep + Split-DP в мультистарте (итоговый выбор)**
  Лучше покрывает разные структуры данных: savings хорошо сшивает "выгодные пары", sweep и split-DP лучше используют геометрию. На практике дает более устойчиво низкую цель.
4. **Локальный поиск только 2-opt без межмаршрутных ходов**
  Быстро, но часто упирается в плохое распределение клиентов между машинами. Добавление relocate/swap улучшает качество.
