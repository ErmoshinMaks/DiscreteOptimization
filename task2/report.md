# Отчёт: задача о рюкзаке (0/1)

## 1. Описание метода

Используется **метод ветвления и границ** (Branch and Bound) с верхней оценкой по **релаксации задачи** (дробный рюкзак).

### Релаксация

В условии разрешено использовать решатель линейного (но не целочисленного) программирования. Используется релаксация: x_i из [0,1]. Такая задача — дробный рюкзак — решается жадно за O(n) после сортировки по убыванию v_i/w_i. Оптимум релаксации даёт верхнюю границу для целочисленного оптимума.

### Алгоритм

1. **Сортировка** предметов по убыванию v_i/w_i — даёт сильную релаксацию и хорошее отсечение.
2. **Начальное решение** — жадное по v_i/w_i: брать предметы целиком, пока помещаются. Это допустимое 0/1-решение и нижняя граница.
3. **Ветвление**: на каждом шаге по текущему предмету рассматриваются две ветви — взять или не взять.
4. **Граница**: для оставшихся предметов и остатка вместимости считается стоимость оптимального дробного рюкзака. Если текущая_стоимость + граница <= best_value, ветвь отсекается.
5. **Обход** — DFS по стеку (LIFO). Состояние: индекс текущего предмета, накопленная стоимость, накопленный вес, вектор взятия по отсортированному порядку.

Сложность в худшем случае экспоненциальная, но на практике сильная релаксация и хорошая начальная нижняя граница сильно сокращают дерево перебора.

## 2. Испробованные улучшения

### Улучшение 1: Best-first (приоритет по границе)

Пробовался обход в порядке убывания верхней границы (приоритетная очередь вместо стека): сначала исследуются наиболее «перспективные» узлы. Ожидался более быстрый набор хорошего решения и усиленное отсечение.

**Результат**: на наших тестах время работы могло увеличиваться из-за накладных расходов на кучу и большего числа одновременно хранимых узлов. Для заданных размеров DFS с жадным стартом оказался выгоднее, поэтому в итоге оставлен стек (DFS).

### Улучшение 2: Динамическое программирование по весу для малых n*W

Для малых произведений n*W (например, n*W < 10^7) пробовалось точное решение методом ДП по весу: состояние — максимальная стоимость при весе не более w. Классическая схема за O(n*W).

**Результат**: на оцениваемых тестах W и n таковы, что n*W велико (например, 10000*1000000), и ДП по весу требовало бы неприемлемой памяти и времени. Для мелких тестов B&B и так быстрый. Отдельное переключение на ДП по размеру входа не вводилось, чтобы не подгонять код под конкретные тесты.

### Улучшение 3: Упорядочивание ветвей (сначала «взять» / «не взять»)

В каждой вершине можно сначала класть в стек ветвь «взять предмет», затем «не взять» — или наоборот. Пробовался вариант «сначала не взять», чтобы быстрее углубляться и обновлять нижнюю границу за счёт жадного продолжения.

**Результат**: на разных тестах выигрывала то одна, то другая стратегия. Оставлен порядок «сначала не взять, потом взять» (ветвь «не взять» кладётся в стек позже и достаётся раньше). Итоговое качество решений не изменилось, порядок выбран из соображений простоты и стабильного времени.
