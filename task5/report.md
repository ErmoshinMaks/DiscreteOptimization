# Facility location

---

## Описание решения

**Идея алгоритма:**

- Считаются расстояния от каждого клиента до каждого магазина.
- Для ускорения локальных улучшений у клиента хранится список **k** ближайших магазинов; на не самых больших размерах в конце возможен полный перебор соседей при переносе.
- **Старт:** жадное назначение при открытых всех магазинах (несколько порядков обхода клиентов); дополнительно пробуется старт с «узкой» маской магазинов с низким отношением **s/cap** (пока суммарная ёмкость не покрывает суммарный спрос с небольшим запасом), если жадность остаётся допустимой и даёт лучшую цель.
- **Улучшение:** перенос одного клиента к более близкому открытому магазину при соблюдении ёмкости; попытки **закрыть** магазин и заново собрать назначение жадно; к концу бюджета времени — случайные попытки закрыть/открыть магазин; **обмен** двух клиентов между разными магазинами, если ёмкости позволяют и сумма расстояний уменьшается.
- Несколько запусков с разными **seed**; общий лимит времени задаётся переменной окружения **FL_TIME**, время делится между запусками; сохраняется лучший результат по цели.



---

## Почему выбран такой метод

Ёмкостная facility location на плоскости с дискретным открытием и назначением — **NP-трудная** задача; точное целочисленное решение на размерах из тестов за разумное время без MIP-солвера недостижимо. Жадная инициализация даёт быстрое **допустимое** решение (при открытых всех магазинах жадность по расстоянию обычно находит назначение, если суммарная емкость достаточна). Локальные шаги (переносы, закрытие магазина с пересборкой, обмены) уменьшают цель, оставаясь в области допустимости. Мульти-старт и разделение времени снижают риск застрять в плохом локальном минимуме одного порядка обхода.

---

## Пробованные улучшения

**1. Порядок обхода клиентов по** разнице между вторым и первым ближайшим расстоянием до магазина, при равной доступности всех точек.

- *Зачем:* сначала «посадить» клиентов, которым некуда уйти, если ближайший магазин заберут по ёмкости.
- *Итог:* на тесных по ёмкости инстансах иногда даёт лучший старт, чем один только порядок по убыванию спроса; на других выигрыш мал или в пределах шума мульти-старта. Стоимость по времени умеренная

**2. Узкая стартовая маска магазинов** (сортировка по **s/cap**).

- *Зачем:* не платить сразу за фиксированные издержки всех кандидатов.
- *Итог:* иногда заметно улучшает начальную цену; но маска может быть **плохой**, жадный назначатель не находит допустимого решения — тогда используется откат к полной маске или к лучшему из сравненных стартов.

**3. Случайные парные обмены двух клиентов между разными магазинами**

- *Зачем:* одиночный перенос одного клиента не всегда выгоден поштучно, а пара «перекинуться» магазинами может уменьшить транспорт без смены набора открытых точек.
- *Итог:* даёт локальные улучшения, которых нет у поштучных переносов; минус — шум от случайных пар и расход времени на больших **M**, поэтому число попыток ограничено.
