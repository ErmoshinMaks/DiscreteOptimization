# Отчёт: домашнее задание 3 (раскраска графа)

## Формат ввода-вывода

**Вход:** первая строка `n m`, далее `m` рёбер `x y` (вершины с нуля).

**Выход (совместимо с `grader.py` из репозитория заданий):**

1. строка: `K 0` — заявленное число различных цветов `K` и флаг оптимальности (`0` = не доказана);
2. строка: `n` целых неотрицательных цветов через пробел;
3. строка: время работы в секундах (float).

Проверка: `python3 verify_output.py <граф> <файл_с_выводом>`.

## Алгоритм

1. **Стартовый портфель:** жадная раскраска в нескольких порядках вершин — Welsh–Powell (по убыванию степени), **DSATUR**, порядок **smallest last** (вывод дегенерации), плюс много случайных перестановок вершин.
2. **Точный поиск для малых графов:** при n≤95 — бинарный поиск по числу цветов и **бэктрекинг** с ограничением на число узлов поиска (случайный tie-breaking и порядок по степени).
3. **Итеративная жадность (Culberson-style):** выбирается случайный цветовой класс, вершины этого класса «стираются» и перекрашиваются в случайном порядке жадно; чередуется с **локальным улучшением** — каждой вершине назначается минимально возможный допустимый цвет (пока улучшается).
4. **Снижение числа цветов:** для целевого k=best−1,best−2,… запускается **локальный поиск по числу конфликтующих рёбер** при палитре {0,…,k−1}: случайная вершина с конфликтом, перебор нового цвета с минимальным Δ по конфликтам; редкий (вероятность 0.004) «подъём» в гору для выхода из плато; инициализации чередуются: **DSATUR с ограничением k**, жадная с ограничением k, случайная, плюс лёгкая мутация.
5. **Бюджет времени:** основная фаза эвристик занимает часть лимита; остаток резервируется под попытки уменьшить k по чанкам (чтобы иметь несколько независимых запусков на одно и то же k).



## Эксперименты с MiniZinc

В файле `coloring.mzn` приведена компактная декларативная модель: `alldifferent` на рёбрах через пары неравенств, целевая `minimize max(c)`. Аннотация `int_search(..., first_fail, indomain_min, complete)` задаёт ветвление, удобное для наблюдения за поведением встроенных эвристик MiniZinc на малых экземплярах. Для реальных тестов курса модель скорее диагностическая: на плотных графах сотен и тысяч вершин полный поиск неприменим, зато на малых \(n\) по ней видно, как влияют порядок переменных и домены.

## Пробовавшиеся идеи (в т.ч. не все дали выигрыш)

- Чистая жадность без DSATUR давала заметно более высокую палитру на плотных инстансах.
- Увеличение числа случайных перестановок в портфеле стабильно улучшало старт.
- Попытка кэшировать множество «плохих» вершин при малом числе конфликтов в локальном поиске по \(k\) оказалась нестабильной (на части запусков ухудшала результат) и была **откатана**.
- Разбиение времени на чанки для повторных попыток уменьшить \(k\) дало более стабильные результаты, чем один длинный прогон до дедлайна.
